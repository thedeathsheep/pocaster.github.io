如果把 2025 年一季度看成「Agent 工程化」的加速期，**MCP** 就是其中最显眼的标准化尝试之一：它承诺用同一套接口描述「模型能调用什么」。热闹之外，三月最值得记笔记的是：**工具一多，上下文与权限就一起爆炸**。

---

## 1. 时事锚点（语境）

- **IDE 与助手生态**密集接入 MCP：开发者第一次能在本地把「私有知识库 + 命令行 + 浏览器」串成一条链。
- **与一月推理模型浪潮叠加**：模型越强，越敢自动多步调用；**运维面风险**随之上升。

---

## 2. 工程 checklist（比「先接十个工具」更重要）

1. **最小权限**
   - 每个 MCP server 声明可读写的资源范围；默认拒绝，再按角色放行。
   - 对写操作加**人工确认或二次令牌**，避免「一句话删库」。

2. **可观测性**
   - 结构化记录：哪个 session、哪步 tool call、入参摘要、耗时、错误类型。
   - 与现有 APM 对齐，而不是另起一套不可查询的日志。

3. **上下文预算**
   - 工具返回的大 JSON 是否全量塞回模型？需要**摘要层**或**分页拉取**策略。
   - 长对话里重复传相同 tool schema 的浪费，可用**会话级缓存**缓解。

4. **供应链**
   - MCP server 也是依赖；要有版本钉扎与漏洞响应流程。

---

## 3. 与「RAG」的关系

- MCP 解决的是**调用面**；向量库解决的是**检索面**。两者可以并存，但不要混为一谈——否则会出现「什么都接进来，但什么都搜不准」。

---

## 4. 本月一句收束

协议统一了插头形状，**没有替代保险丝**；三月适合给 MCP 接线前先画权限矩阵。

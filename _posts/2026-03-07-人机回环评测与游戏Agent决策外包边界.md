游戏里谈 Agent，早就不限于「NPC 多几句台词」。我见过的发展方向包括：根据存档推荐下一步任务、根据构筑提示技能加点、在 UI 里一键整理背包、甚至在测试服里扮演「陪练 bot」。这些能力的共同点是：**系统开始替玩家做一部分决策**。

决策外包一旦越界，体验会从「省事」滑向「被教玩游戏」或「被悄悄改玩法」。所以我更在意的不只是模型话术，而是**外包边界是否写清楚、是否可评测、是否可回滚**。

![Human in the loop](/assets/img/roadmap.jpeg)

## 1. 三种「外包」强度：建议、半自动、全自动

### 1.1 建议型（advisory）

只生成文本或高亮提示，不直接改游戏状态。风险相对低，但仍有「信息污染」：错误建议会浪费注意力。

### 1.2 半自动（assisted execution）

需要玩家确认后才执行（例如「是否自动寻路」「是否一键合成」）。关键是**确认 UI 是否足够显著**，以及默认选项是否操纵性强。

### 1.3 全自动（autonomous）

在规则允许范围内直接改状态（例如自动拾取过滤、自动战斗脚本）。这是事故高发区：**玩家未必意识到状态已被改变**。

我会要求产品文档里显式标注当前功能属于哪一档，而不是混用「智能助手」这种模糊词。

## 2. 人机回环：不是「有人盯着」就叫回环

真正的人机回环需要**可追溯的干预记录**：

- 模型提出了什么行动提案（proposal）？
- 玩家是接受、修改还是拒绝？
- 若拒绝，后续模型是否学到约束（在同一 session 或长期偏好里）？

没有记录，就只能凭客服工单猜问题。

## 3. 评测维度：除了「像不像人」

### 3.1 任务完成增益（utility delta）

对比有/无 Agent 时，**关键漏斗**是否改善：新手卡点通过率、任务放弃率、平均迷茫时长（例如无目的漫游时间）。

### 3.2 越权率（overreach rate）

统计提案中违反设计规则的比例：建议装备与当前关卡不匹配、推荐未解锁内容、引导至已废弃支线等。这类错误对信任伤害极大。

### 3.3 干扰度（intrusion）

弹窗频率、强制打断次数、占用屏幕面积时长。再强的模型，若总在错误时机说话，也会被关掉。

### 3.4 一致性（consistency）

同一存档、同一问题，多次询问是否给出自相矛盾建议？若存在随机性，是否需要降温度或引入检索记忆？

## 4. 离线回放：用「固定存档」做回归

我会维护一组**黄金存档**（golden saves）：覆盖新手、中期构筑、边缘分支、经济崩溃边缘等。对每个存档跑固定问题集与固定 UI 路径，比较：

- 提案结构是否仍合法；
- 关键建议是否漂移；
- 文案是否触发新的合规风险。

这与传统软件里的 golden file testing 同类：**不是为了证明好玩，而是为了证明没有悄悄变坏**。

## 5. 线上指标：把「关掉助手」当成信号

除了点击率与使用时长，我会看：

- **关闭率与永久关闭率**：是否在特定关卡后飙升？
- **否决率**：半自动场景里玩家取消确认的比例。
- **纠错后成功率**：玩家编辑计划后是否更愿意继续？

这些指标比「模型评分」更接近真实满意度。

## 6. 与游戏经济、公平性的交界

若 Agent 能影响资源获取（自动合成、自动出售、自动刷本策略），就可能触及**经济与工作量的公平**。我倾向于：

- 对外挂与脚本已有底线的地方，Agent 不能「合法化绕过」；
- PVP 场景默认禁用或极度收缩能力；
- 所有自动收益路径可审计。

这不是模型问题，是**产品伦理与规则设计**问题，但要写进需求与评测项，否则技术团队会被动背锅。

## 7. 落地检查清单（节选）

- [ ] 功能档位（建议/半自动/全自动）已写明
- [ ] proposal / intervention 日志字段齐全，可关联 trace
- [ ] 黄金存档回归每周至少跑一轮
- [ ] 越权类错误有明确降级策略（拒绝执行 + 解释）
- [ ] 「关闭助手」与「否决」进入仪表盘

## 8. 收束

Agent 在游戏里最容易犯的错误不是「不够聪明」，而是**在不该外包的时刻外包了决策**。人机回环的价值，是把外包变成可见、可撤销、可统计的过程。我的实践里，只要把边界与日志先立住，模型迭代反而可以更快——因为终于能分清「话术问题」与「权力越界问题」。
